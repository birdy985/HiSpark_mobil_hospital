# Hospital patient info sync UI

## Goal

在现有云端生命体征监测链路已经打通的基础上，提升电脑端和手机 H5 页面的医院场景感，并新增患者信息录入页面/区域。电脑端、手机端、EMQX、WS63 服务端板子需要围绕同一份患者信息实时同步，板子接收到后再通过串口屏显示患者信息。

## Confirmed Facts

- 当前前端在 `web-dashboard/`，由 `index.html`、`mobile.html`、`styles.css`、`dashboard.js` 组成，是纯静态页面。
- 当前前端通过 MQTT.js 连接 EMQX WebSocket，订阅 `hispark/bed01/telemetry` 展示 ECG、心率、血氧、质量、原始消息。
- 当前板端 MQTT 逻辑在 `fbb_ws63/src/application/samples/mydemo/15_emqx_telemetry/emqx_telemetry_mqtt.c`，使用 TLS 连接同一 EMQX，并每秒向 `hispark/bed01/telemetry` 发布生命体征遥测。
- 当前板端 MQTT 逻辑没有订阅患者信息 topic，也没有解析网页下发的患者信息。
- 当前 SLE 服务端板子串口屏逻辑在 `fbb_ws63/src/application/samples/mydemo/13_ad8232_sle/tjc_display.c`，已封装 TJC 串口指令发送，当前只显示 ECG 波形和 BPM。
- `tjc_display.c` 使用 UART2，TX=GPIO08，RX=GPIO07，115200 baud。
- 串口屏控件名已由用户确认：`t0` 心率、`t1` 心电实时值、`t2` 血氧、`t3` 心率、`t4` 收缩压、`t5` 舒张压、`t6` 微循环、`t7` 姓名、`t8` 病历号、`t9` 性别、`t10` 年龄、`t11` 联系电话、`t12` 备注。

## Requirements

- 页面视觉需要从当前基础监测面板调整为更符合医院/病区风格的界面。
- 电脑端和手机端都需要支持录入并提交以下患者字段：
  - 病人姓名
  - 病历号
  - 性别
  - 年龄
  - 联系电话
  - 备注
- 电脑端提交患者信息后：
  - 电脑端本页显示最新患者信息和提交状态。
  - 手机端能收到并显示同一份最新患者信息。
  - WS63 服务端板子能收到同一份患者信息。
  - 板子能把患者信息转发给串口屏显示。
- 手机端提交患者信息后：
  - 手机端本页显示最新患者信息和提交状态。
  - 电脑端能收到并显示同一份最新患者信息。
  - WS63 服务端板子能收到同一份患者信息。
  - 板子能把患者信息转发给串口屏显示。
- 患者信息同步不能破坏现有 `hispark/bed01/telemetry` 遥测显示链路。
- 不要修改其他已验证功能；执行时只能做医院风格 UI、患者信息同步、板端接收和串口屏患者字段显示所需的最小改动。
- 串口屏患者字段必须写入 `t7` 至 `t12`；不得改变 `t0` 至 `t6` 已有/预留生命体征控件的语义。
- 前端仍应保持静态部署方式，不引入必须单独运行的 Web 后端服务。
- 患者信息 MQTT payload 需要包含来源、序号或时间戳，方便电脑端/手机端/板端判断最新数据和排查链路。
- 输入校验需要避免明显无效数据：
  - 姓名、病历号为必填。
  - 性别使用固定选项。
  - 年龄为合理数字。
  - 电话允许为空，但填写时应限制为电话格式友好的字符。
  - 备注限制长度，避免 MQTT payload 和串口屏命令过长。

## Acceptance Criteria

- [ ] `web-dashboard/index.html` 页面呈现医院病区监测风格，并保留 ECG、心率、血氧、质量、连接状态、原始消息等现有功能。
- [ ] `web-dashboard/mobile.html` 页面呈现与电脑端一致的医院风格，并适配手机窄屏操作。
- [ ] 电脑端和手机端都能录入姓名、病历号、性别、年龄、联系电话、备注并提交。
- [ ] 任一端提交后，发布到患者信息 topic 的 JSON 至少包含 `type`、`deviceId`、`source`、`seq`、`ts`、`patient` 字段。
- [ ] 电脑端提交患者信息后，手机端无需刷新即可更新最新患者信息显示。
- [ ] 手机端提交患者信息后，电脑端无需刷新即可更新最新患者信息显示。
- [ ] WS63 服务端板子订阅患者信息 topic 后，串口日志能打印收到的患者信息摘要。
- [ ] WS63 服务端板子收到患者信息后，调用 TJC 串口屏接口更新 `t7` 姓名、`t8` 病历号、`t9` 性别、`t10` 年龄、`t11` 联系电话、`t12` 备注。
- [ ] 现有 `hispark/bed01/telemetry` 遥测 topic 仍能正常驱动电脑端和手机端生命体征显示。
- [ ] README 更新电脑端、手机端、MQTTX、WS63、串口屏的验证步骤。

## Out of Scope

- 不新增数据库或长期病历存储。
- 不实现登录、权限系统或患者隐私加密存储。
- 不改变 ECG/SLE/SpO2 采样算法。
- 不重做串口屏 HMI 工程文件；只按用户确认的 TJC 控件名发送文本更新指令。
- 不改变当前 EMQX 云服务地址和现有遥测 topic，除非执行时发现当前配置无法同时支持订阅。
